## 여는 글

자료구조를 공부하다가 `python`의 `deque` 라는 자료구조를 알게됐다.

`deque`는 Double Ended Queue 의 약자로 양쪽에서 삽입, 삭제가 가능한 큐(Queue) 형태의 자료구조로
큐를 `list`로 구현해 삭제(`pop()`)하는 시간 복잡도는 O(n)인 반면에, `deque`의 경우 삭제 `popleft()`하는 시간 복잡도는 O(1) 이라고 한다.

결과만 보면 같은 동작을 하는것 같은데 다른 시간 복잡도를 가지는게 신기해서, `deque`가 메모리 상에서 어떻게 동작하는지 알아보았다.

## 큐 구현하기

First In First Out인 큐의 삽입과 삭제를 `list`로 구현해보자.

먼저 `list`를 큐로 선언하고 `append()` 메서드를 통해 원소를 삽입할 수 있다.

```python
queue = []

queue.append(3)  # [3]
queue.append(4)  # [3, 4]

# ...
queue.append(n)  # [3, 4, ... , n]
```

같은 형태의 큐를 `deque`로 작성하면 다음과 같다.

`deque`를 가져와 선언해준뒤 마찬가지로 `append()` 메서드를 통해 원소를 삽입해줬다.

```python
from collections import deque

dq = deque()

dq.append(3)  # [3]
dq.append(4)  # [3, 4]

# ...
dq.append(n)  # [3, 4, ... , n]
```

여기까지 `list`와 `deque`로 구현된 큐의 형태는 다를게 없어 보이지만 메모리의 구조를 비교하면서 시간복잡도가 달라지는 이유를 살펴보자.

그렇다면 `deque`의 경우는 어떨까?

## deque의 원소 제거

`deque`는 내부적으로 고정된 크기를 가지는 블록을 이중 연결 리스트처럼 연결해 구성된다.

각 블록은 여러 요소를 저장할 수 있는 슬롯을 가지고 있고, `head`와 `tail` 두 개의 포인터가 요소가 위치한 범위를 추적한다.
현재 블록의 포인터가 다음 블록의 주소를 기억하고 있어 블록의 슬롯이 전부 채워진 경우 다음 블록으로 이동하게 된다.

3 부터 n 까지의 원소를 삽입한 후의 메모리상 구조는 다음과 같다.

<p> </p>

> [!note] tail의 위치?
> `tail` 포인터는 마지막 원소의 위치가 아니라 그 다음에 삽입 가능한 빈 슬롯의 위치를 가리킨다.
> 다음 메모리 슬롯을 가리키고 있어 새로운 원소가 들어오면 해당 주소에 저장한다.

<p> </p>

```python
from collections import deque

dq = deque([3, 4, 5, ... , n])
dq.popleft()  # 3 제거, [4, 5]
```

연결된 블록에 데이터를 각각 나눠 담은 뒤, `popleft()`를 통해 앞에서 요소를 제거하면 어떻게 되는지 살펴 보자.

<MDXImage
  src="posts/deque/deque-popleft.png"

  caption="포인터만 이동하여 요소를 제거하는 deque의 popleft"
/>

`deque`의 메서드 `popleft()`는 내부적으로 요소를 이동시키는 것이 아니라 `head` 포인터를 뒤로 옮기기 때문에 연산의 시간 복잡도가 `O(1)`로 `list`로 구현한 큐에 비해 매우 효율적이다.

추가로 포인터의 영역에서 벗어나게 된 값은 메모리에 그대로 남아 있게 되는데 참조 카운트가 0이 되거나 `deque` 내부에서 이 슬롯을 재사용할 때 메모리에서 정리되게 된다.

## 닫는 글

정리하면 `list`와 `deque`를 이용해 큐를 구현했을 때의 차이는 다음과 같다.

> `list`는 연속된 메모리 공간에 저장하기 때문에 앞에서 요소를 제거할 경우 모든 요소를 한 칸씩 이동해야 한다.
> 시간복잡도 : `O(n)`

<p> </p>

> `deque`는 고정된 크기의 블록을 연결한 구조로 이루어져 있고 요소 제거 시 포인터만 이동한다.
> 시간복잡도 : `O(1)`

겉으로 보기엔 동일한 동작처럼 보이지만 자료구조에 따라 성능, 시간 복잡도가 달라지게 된다.
자료구조를 사용할때 어떤 방식으로 구현되어 있는지 이해하고 사용할 수 있도록 정리해야겠다.
