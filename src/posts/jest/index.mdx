저에게 테스트 코드란 항상 어려운 부분이었어서 이번 기회에 제대로 공부하려고 해요.
프로젝트가 커질수록 테스트 코드의 중요성은 더욱 커지는데, TDD 처럼 테스트를 위주로 한 개발 방법론도 있는걸 보면 테스트는 중요한 부분이라고 볼 수 있을것 같아요.

여러 프론트엔드 테스팅 라이브러리가 있지만 그 중에서도 많이 쓰는 라이브러리로 Jest, Vitest 가 있어요.
요즘에 많이 사용하는 추세인 Vitest도 Jest 문법을 기준으로 하기 때문에, Jest를 먼저 공부 해보도록 할게요.

이 글은 [공식문서](https://jestjs.io/)와 [제로초님의 Jest 강의](https://inf.run/3Veyh)를 바탕으로 정리했어요.

먼저, 다음과 같은 기준을 세우고 정리해보려고 했어요.

- 테스트 코드를 틈틈이 작성할 수 있도록 친숙해지자.
- 테스트를 위한 테스트가 아니라 의미있는 테스트 코드를 작성해보자.
