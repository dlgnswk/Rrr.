## 여는 글

프론트엔드 개발에서 일정 관리에 어려움을 겪는 이유중에 하나로 API 개발 일정 지연을 뽑을 수 있는데, 그 이유는 API 개발 일정과 밀접하게 연관되어 있기 때문일 거에요.
현업에서 프론트엔드 개발하면서 일정 지연을 경험했던 케이스들은 이런 것들이 있었어요.

- 정적인 UI 구현을 완료한 시점에 필요한 API 개발이 되지 않은 경우
- 개발된 API로 특정 기능을 구현할 때 API에서 에러가 발생하는 경우
- 개발하는 도중에 API 스펙이 변경되는 경우
- 등등...

이런 상황에서 더 효율적인 개발, 협업을 가능하게 해주는 API 모킹(mocking) 라이브러리인 `Mock Service Worker, MSW`에 대해 알아볼게요.

`MSW`는 `Service Worker`를 사용해서 네트워크 수준에서 API 요청을 가로채는 API 모킹(mocking) 라이브러리예요.
기존의 모킹 방식과 다르게 실제 네트워크 요청을 차단하고 모의 응답을 반환하기 때문에, 마치 실제 서버와 통신하는 것처럼 개발을 진행할 수 있어요.

> [!note] Service Worker?
> 서비스 워커는 연관된 웹 페이지/사이트를 통제하여 탐색과 리소스 요청을 가로채 수정하고, 리소스를 굉장히 세부적으로 캐싱할 수 있습니다. 이를 통해 웹 앱이 어떤 상황에서 어떻게 동작해야 하는지 완벽하게 바꿀 수 있습니다. (제일 대표적인 상황은 네트워크를 사용하지 못할 때입니다.)
> [MDN Service Worker API](https://developer.mozilla.org/ko/docs/Web/API/Service_Worker_API)

주요 특징으로는:

- 브라우저와 Node.js 환경 모두 지원
- REST API와 GraphQL 요청 모두 모킹 가능
- 실제 네트워크 요청과 동일한 개발 경험

## 닫는 글

효율적인 개발을 위해 MSW를 프로젝트에 도입해보았는데 겪었던 일정관리에 대한 불편을 줄여줄 뿐만 아니라, 개발 경험에서도 여러가지 측면이 개선되었다는 생각이 들었어요.

- API 개발 작업이 선행되지 않아도 병렬적으로 프론트엔드 개발 진행 가능
- 다양한 API 응답 상황에 대한 테스트 가능
- 백엔드 서버 상태와 무관한 개발 환경 유지
